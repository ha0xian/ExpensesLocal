# Repository Instructions

## Automatic Publishing

- After completing and verifying any code change in this repository, automatically commit the files belonging to that change and push the commit directly to `origin/main` without waiting for a separate request.
- Before committing, confirm that the working tree contains no unrelated changes. Never stage, commit, overwrite, or revert unrelated user work.
- Synchronize with `origin/main` before pushing. If synchronization would require resolving conflicts, or if verification, committing, or pushing fails, stop and report the problem instead of forcing the operation.
- Use a concise commit message that describes the completed change.
- This instruction applies to implementation changes. Do not create commits for read-only investigation, planning, review, or explanation-only tasks unless explicitly requested.
